# 🏢 公司内部任务系统 v2.0

一个支持多用户协作的任务管理系统，专为公司内部使用设计。

---

## ✨ 核心功能

### 👔 老板/管理员界面
- **专属后台**：独立的管理界面，与员工界面完全不同
- **成员管理**：实时查看所有注册员工
- **任务概览**：查看所有任务，按员工/状态筛选
- **发布任务**：设置标题、描述、优先级、截止时间，指派员工
- **取消任务**：随时取消不需要的任务
- **实时监听**：员工注册后自动更新员工名单（1秒轮询）

### 👤 员工界面
- **账号注册**：姓名 + 账号 + 密码
- **账号登录**：使用注册的账号密码登录
- **专属任务列表**：只看到指派给自己的任务
- **完成按钮**：点击标记任务完成，自动同步到老板界面
- **状态提示**：逾期任务紫色警告

### 🎨 状态颜色
| 状态 | 颜色 | 说明 |
|------|------|------|
| 已完成 | 🟢 绿色 | 任务已完成 |
| 未完成 | 🔴 红色 | 任务进行中 |
| 已逾期 | 🟣 紫色 | 超过截止时间未完成 |

### 🔄 实时同步
- 1 秒轮询，实现准实时效果
- 员工注册 → 老板界面自动更新员工名单
- 员工完成任务 → 老板界面自动更新任务状态
- 所有数据保存在 localStorage（本地模式）

## 🚀 使用方式

### 方式一：直接打开（本地模式）

双击 `index.html` 即可使用，数据保存在浏览器本地。

**管理员登录**：
- 密码：`admin123`

**员工使用**：
1. 选择「员工」身份
2. 点击「注册账号」
3. 填写姓名、账号、密码
4. 注册成功后登录

### 方式二：Docker 部署（多设备同步）

```bash
docker-compose up -d
# 访问 http://localhost
```

## 📌 数据结构

### 员工
```javascript
{
  id: String,
  name: String,        // 姓名
  username: String,    // 账号
  password: String,    // 密码
  createdAt: String
}
```

### 任务
```javascript
{
  id: String,
  title: String,
  description: String,
  priority: 'high' | 'medium' | 'low',
  deadline: String,      // 截止时间
  assigneeId: String,    // 指派员工 ID
  assigneeName: String,
  status: 'pending' | 'completed',
  updates: [],           // 更新记录
  createdAt: String,
  completedAt: String
}
```
